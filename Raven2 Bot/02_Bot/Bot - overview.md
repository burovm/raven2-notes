## Project Goal

Raven2 Auction Bot — это инструмент для анализа и автоматизации торговли террамондами в игре Raven 2.  
Бот собирает историю цен с аукциона, находит выгодные моменты для покупки и продажи и может:  
- работать как ассистент (давать сигналы игроку);  
- или в авто-режиме имитировать действия игрока и сам совершать сделки.

## Key Features (planned)

### 1. Data Collection

- Периодический сбор данных с аукциона террамондов (через скриншоты/захват окна клиента).  
- Парсинг лотов:
  - тип террамонда (item_type);
  - цена (price);
  - количество (quantity);
  - время снятия данных (timestamp);
  - направление сделки (side: buy/sell);
  - дополнительные поля (seller, extra), если доступны.
- Сохранение данных в локальную БД SQLite (`data/raven2.db`, таблица `lots`).

### 2. Analytics & Strategy

- Расчёт статистики по каждому типу террамонда:
  - средние цены;
  - минимумы/максимумы;
  - изменение цен по дням недели и времени суток.
- Поиск паттернов:
  - периоды, когда цена обычно ниже (окна для закупки);
  - периоды, когда цена выше (окна для продажи).
- Настраиваемые стратегии:
  - фильтрация по типу террамонда;
  - минимальная маржа;
  - лимиты по объёму/риску.

### 3. Assistant Mode

- Генерация рекомендаций:
  - какие террамонды покупать сейчас, по какой цене и в каком объёме;
  - какие террамонды продавать сейчас и по какой цене.
- Вывод сигналов:
  - в лог/консоль;
  - в файлы;
  - (в будущем) в Telegram/Discord-бота для уведомлений.

### 4. Auto-Execution Mode

- Модуль автокликера, который:
  - открывает аукцион в клиенте;
  - находит нужный террамонд;
  - покупает/выставляет лоты по рассчитанным параметрам.
- Имитация человеческого поведения:
  - случайные задержки и вариации движений;
  - ограничение времени работы и частоты операций;
  - приоритет безопасного поведения, чтобы снизить риск бана.

### 5. Tooling & Observability

- Инспекция БД:
  - просмотр последних лотов;
  - простые отчёты по ценам и объёмам.
- Логи действий бота:
  - какие сигналы были сгенерированы;
  - какие сделки были совершены в авто-режиме.
- Быстрый переключатель:
  - режим только-аналитика/ассистент;
  - режим с автоматическим исполнением сигналов.


## Архитектура Raven 2 bot (2026-03-09)

- Proxmox / rocketsrv
  - Хост-гипервизор, на котором живут все VM проекта.
  - Включен IOMMU, настроен GPU passthrough для одной RTX 4090 под Win11-VM.
  - Основные VM:
    - `bot-debian` – backend и аналитика.
    - `win-raven2` – Windows 11 с Raven 2 + OCR-клиент (GPU passthrough 4090).

- VM `bot-debian` (Linux backend)
  - Репозиторий `raven2-bot` в `~/projects/raven2-bot`.
  - Стек: Python + FastAPI + SQLAlchemy + SQLite.
  - Модели БД:
    - `terramonds` (id, code, name, ...),
    - `snapshots` (id, taken_at, source, ...),
    - `lots` (id, snapshot_id, terramond_id, unit_price, quantity, direction).
  - API:
    - `POST /snapshots` – принимает JSON с `taken_at`, `source`, `lots[]` (lot без `terramond_code`, только `terramond_name` и числа), автосоздаёт террамонды по имени.
    - Набор эндпоинтов статистики и сигналов по отдельным террамондам.
  - Запуск:
    - Docker + docker-compose, контейнер `raven2-backend` на порту 8000.

- VM `win-raven2` (Windows 11 + Raven 2)
  - Windows 11 Pro/Enterprise (VM в Proxmox, q35 + OVMF + CPU host).
  - GPU passthrough RTX 4090 (PCI 05:00.0/05:00.1), драйвер NVIDIA установлен.
  - ПО:
    - Python 3.14 + pip, директория `C:\raven2-bot`.
    - Клиент `fake_client.py` – периодически шлёт тестовые снапшоты в backend `/snapshots`.
    - Epic Games Launcher + клиент Raven 2; игра работает и при закрытом RDP.
  - Будущая роль:
    - Запуск клиента Raven 2.
    - Снятие скринов экрана террамондов.
    - OCR + парсинг в структуру `lots[]` и отправка в backend.

- Общее взаимодействие
  - Win11-VM → backend:
    - HTTP `POST /snapshots` с JSON:
      - `taken_at` (UTC),
      - `source` (идентификатор клиента/VM),
      - `screen_type` (например, `market_terramonds`),
      - `lots[]` (имя террамонда, количество, цена за единицу, направление).
  - Backend → SQLite:
    - Сохраняет все снапшоты и лоты, строит агрегаты и сигналы.
  - Obsidian:
    - Vault `C2 Bot` описывает инфраструктуру (Proxmox, VM), backend (модели/эндпоинты), Windows-VM и OCR-пайплайн, плюс ежедневный журнал прогресса.
