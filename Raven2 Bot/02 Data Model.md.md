# Data Model

## База данных

На первом этапе используется SQLite, локальный файл БД на сервере rocketsrv (например, `~/projects/raven2-bot/data/raven2.db`). В дальнейшем при необходимости можно перейти на PostgreSQL.

## Таблица `lots`

Основная таблица для хранения лотов террамондов с аукциона:

- `id` — INTEGER, PRIMARY KEY, AUTOINCREMENT
- `timestamp` — DATETIME, когда был снят снимок лота (время считывания с аукциона)
- `server` — TEXT, сервер/регион (например, EU Tempus / другой, который ты используешь)
- `item_type` — TEXT, тип террамонда (название/категория)
- `price` — INTEGER или REAL, цена за единицу (в игровой валюте)
- `quantity` — INTEGER, количество в лоте
- `side` — TEXT, направление: 'buy' или 'sell' (покупка или продажа, если UI разделяет)
- `seller` — TEXT, имя продавца (если доступно и имеет смысл сохранять)
- `extra` — TEXT/JSON (опционально), дополнительные данные, если появятся

## Milestone 1: Data collector MVP

Задача:
- Научиться с определённой периодичностью снимать состояние аукциона по террамондам.
- Из каждого снимка извлекать список лотов и записывать их в таблицу `lots`.

Минимально необходимые поля для MVP:
- `timestamp`
- `server`
- `item_type`
- `price`
- `quantity`
