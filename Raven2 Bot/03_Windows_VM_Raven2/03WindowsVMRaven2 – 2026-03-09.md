# Windows VM Raven 2 – состояние на 2026-03-09

## 1. Назначение VM
- Эта Win11 VM в Proxmox – рабочее место бота-наблюдателя за Raven 2.
- Здесь крутятся: клиент игры, OCR, Python-скрипты, которые отправляют сделки в backend на `bot-debian`.

## 2. Параметры VM
- Гипервизор: Proxmox (узел rocketsrv).
- Имя VM: <имя_из_Proxmox> (уточнить).
- Ресурсы: CPU/ RAM / диск (заполнить по факту).
- Сеть:
  - Тип адаптера: VirtIO.
  - IP Win11: <ip_win11>.
  - IP `bot-debian`: 192.168.50.185 (пример, уточнить).
  - Пинг Win11 → `bot-debian`: работает.

## 3. Установка и базовая настройка
- Установка:
  - ISO Windows 11, VirtIO-драйверы подключены как отдельный CD.
  - В процессе установки подгружены драйверы диска/сети с `virtio-win` ISO при необходимости.
- Драйверы:
  - Установлен пакет `virtio-win-gt-x64` (virtio guest tools).
  - После установки появился рабочий сетевой адаптер, Win11 получает IP по DHCP.
- Базовые настройки:
  - Включён RDP-доступ.
  - Отключён автоматический сон и отключение экрана для стабильной работы бота.
  - Настроены часовой пояс и автообновления (уточнить, как именно).

## 4. Python-окружение
- Версия Python:
  - Установлен Python 3.14.3 (официальный дистрибутив для Windows).
  - Проверка: `python --version` → `Python 3.14.3`, `pip --version` → актуальная.
- PATH:
  - Python и pip доступны из обычного CMD без дополнительных танцев.
- Пакеты:
  - Установлен `requests` (через `pip install requests`).
  - Планируются: `opencv-python`, `pillow`, библиотеки для OCR (уточнить после выбора стека).

## 5. Структура рабочих каталогов на Win11
- Рабочие папки:
  - `C:\raven2-bot\` – основной каталог для Windows-клиента.
  - Файлы:
    - `test_send_snapshot.py` – тестовый скрипт HTTP-клиента (Win11 → backend).
    - Далее: основной скрипт OCR-клиента, конфиги и логи.
- Обсуждаемый формат JSON для отправки:
  - `taken_at`: UTC datetime.
  - `source`: строка с идентификатором клиента/VM (например, `winvm1-client1`).
  - `lots`: массив объектов:
    - `terramond_code`
    - `terramond_name`
    - `unit_price`
    - `quantity`
    - `direction` (`buy` / `sell`)
