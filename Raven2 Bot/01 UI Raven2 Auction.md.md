# Architecture

## Сервер и путь проекта

- Сервер: rocketsrv
- Путь проекта: `~/projects/raven2-bot`

## Структура папок

Планируемая структура:

- `src/ravenbot/` — основной пакет проекта
  - `src/ravenbot/ui_capture/` — захват экрана/интерфейса аукциона и извлечение данных лотов
  - `src/ravenbot/data/` — работа с БД (подключение, миграции, CRUD по лотам)
  - `src/ravenbot/analysis/` — анализ цен, поиск недельных паттернов, генерация сигналов
  - `src/ravenbot/executor/` — автокликер/исполнитель сделок по сигналам
- `configs/` — конфигурационные файлы (пути, параметры БД, настройки UI)
- `scripts/` — вспомогательные скрипты запуска (например, `run_collector.sh`)
- `tests/` — тесты для ключевых модулей (по мере появления кода)

## Роли модулей

- **ui_capture**: получает изображения аукционного окна, извлекает список лотов террамондов (цена, количество, прочее) и передаёт их в слой data.
- **data**: хранит данные о лотах в БД (на первом этапе SQLite), предоставляет удобный интерфейс для чтения/записи.
- **analysis**: работает с историей цен, считает агрегаты по дням/неделям, определяет выгодные моменты для покупки/продажи.
- **executor**: управляет мышью/клавиатурой, имитирует человекоподобные действия в игре и исполняет торговые решения с учётом ограничений безопасности.
